## 🍀 청계산 옥녀봉 등산 일기

> 등산은 밥을 먹고! 가방은 가볍게!

&nbsp;회사 동료분들이 등산을 하신다고 해서, '재밌는 얘기하는 곳에 빠질 수 없지!' 마인드로 참여한 등산. 등산을 한 게 언제였더라.. 한 5년 정도 된 것 같다. 그렇게 젊을 때도 헥헥거리면서 힘들어했었는데 고새 까먹고 또 도전을 한 거지.

&nbsp;아침에 아침밥을 먹고 갈 생각으로 일찍 알람을 맞춰놨었는데, 미적대다 보니까 밥을 못먹어서 에너지바랑 과자를 챙겼다. 몰랐는데 등산고수 친구가 등산 갈 때는 꼭 밥을 먹어야한다고 하더라. 역시 모르는 자가 용감하다. 보시는 분들은 꼭 밥을 먹고 가도록 하세요.

&nbsp;아무튼 도착해서 `매봉`으로 올라가는데 진심 힘들어서 머리가 띵했다. 내가 나이는 제일 젊었는데..! 엄청난 사람들 어떻게 그렇게 체력이 좋으신지. 개발도 잘하시고 운동도 잘하시고, `개발자하려면 이정도는 기본인가?`

&nbsp;그래서 나때문에 `옥녀봉`으로 목표를 바꿔서 쭉 올라갔다. 체력왕님이 가방을 들어주셔서 살았다(감사합니다.. 은혜는 꼭 갚겠읍니다..) 경치 넘 좋더라~ 산에 왜 가는지 알 거 같은 느낌. 체력 좀 키워야겠다는 생각이 많이 들었다.

&nbsp;올라갈 때는 울퉁불퉁한 돌 계단이 너무 많아서 바닥만 보면서 올라갔는데, 내려갈 때는 제법 여유가 생겨서 나무도 구경하면서 담에 또 오고 싶다는 생각이 들었다. 으른들이 등산을 좋아하는 데는 다 이유가 있다. 건강해지는 느낌. 강추!

## 🍁 개발 용어 정리

&nbsp;최근에 주워들은 용어를 정리해보는 시간.

&nbsp;어디선가 들어본 적이 있어도 뭔지 기억이 안 날 때가 있다. 특히 `약어`! 나는 약어는 풀네임을 기억해야 어떤 내용이었는지 기억이 나더라. 근데 내가 직접 써보지 않은 것들은 아무래도 잘 까먹게 된다.

&nbsp;그래서 잘 정리해두면 나중에 까먹었을 때 내가 꺼내보기 좋을 거 같다. 잘 정리해놓으신 개발자분들 존경합니다(자세한 내용은 출처 참고)

- 모놀리식 아키텍처(Monolithic Architecture, MA)
  - 모든 비즈니스 관련 사항을 함께 결합하는 하나의 코드 베이스를 갖춘 대규모의 단일 컴퓨팅 네트워크
  - 장점: 배포/테스트 간단
  - 단점: 확장성, 안정성
- 마이크로서비스 아키텍처(Microservice Architecture, MSA)
  - 애플리케이션을 상호 독립적인 최소 구성 요소로 분할
  - 장점: 확장성, 서비스별 배포 가능
  - 단점: 트랜잭션/통신 비용 발생, 개발 시간 증가
- git 체리픽(cherry-pick)

  - A 브랜치에서 필요한 커밋만 가져다가 B 브랜치에 적용하는 방법

    ```
    git cherry-pick <커밋번호>
    ```

    체리픽이 `케이크 위에 얹어져있는 체리를 쏙 빼가는 행위`에서 나왔다는 얘기를 듣고 너무 인상 깊었다 ㅋㅋㅋ

- ERD(Entity Relationship Diagram)

  개체-관계 모델. 시스템의 엔티티들이 무엇이 있는지, 어떤 관계를 가지는지 나타내는 다이어그램

- JPA(Java Persistence API)
  - Java 진영에서 ORM(Object-Relational Mapping) 기술 표준으로 사용하는 인터페이스 모음
  - 자바 어플리케이션에서 관계형 데이터베이스를 사용하는 방식을 정의한 인터페이스
- ORM(Object-Relational Mapping)

  애플리케이션의 객체를 RDB(Relational DataBase)에 자동으로 매핑
